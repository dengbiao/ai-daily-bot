# 每日AI日报系统需求文档

## 项目目标
构建一个自动化每日AI日报系统，部署在GitHub Pages上，内容严谨且所有信息可追溯原始信源。

## 功能需求

### 1. 数据采集
- **AI论文**: 从 HuggingFace Daily Papers 获取最新热门论文
- **开源项目**: 从 GitHub Trending 获取当日热门AI相关仓库
- **行业新闻**: 从优质AI媒体RSS源（如 MIT Technology Review、KDnuggets等）获取新闻

### 2. 内容格式
- 每条信息以**卡片形式**展示
- 每条信息必须包含：标题、摘要/简介、可点击的原始来源链接（保证可追溯）
- 分类展示：📄论文 / ⭐开源项目 / 📰新闻

### 3. 展示与部署
- 部署到 GitHub Pages，支持自定义域名或子路径
- **响应式设计**：完美适配桌面、平板、手机
- 首页显示文章列表（每天一篇日报）
- 详情页展示当天所有卡片内容
- URL路径**不要有中文**，使用英文（如 `/daily-ai-news/`）

### 4. 自动化
- 每天定时自动运行（北京时间每天早上8点）
- 自动抓取数据、生成日报、提交到GitHub、自动部署
- 支持手动触发更新

## 技术要求
- 使用 Jekyll + GitHub Pages 构建（可选其他静态生成器）
- 可以使用现成的主题（如 Chirpy），但要确保能正确显示和自定义布局
- Python脚本用于数据采集，生成Markdown格式日报
- GitHub Actions 用于自动化部署

## 验收标准
- [ ] 首页能正常显示文章列表
- [ ] 点击文章进入详情页，能看到卡片式布局的内容
- [ ] 每条内容都有可点击的原始链接
- [ ] 页面在手机上显示正常
- [ ] 每天自动更新新内容
- [ ] URL中没有中文字符

## 当前问题
1. 主题配置复杂，Chirpy本地部署需要完整的主题文件和插件
2. 需要正确的 `include_cached` 插件支持
3. 路径和URL编码问题需要避免中文
4. 需要完整的布局文件（_layouts, _includes, _sass 等）

## 建议简化方案
考虑到实现复杂度，建议：
1. 使用更简单的主题或自定义布局，而不是完整Chirpy
2. 直接使用 HTML + Tailwind CSS 构建简洁的卡片式布局
3. 确保GitHub Pages能正确渲染，不依赖复杂的主题配置
